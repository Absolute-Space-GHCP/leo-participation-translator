## Future Considerations

### Technical Enhancements

1. **Vertex AI Vector Search Migration**
   - Currently using Firestore for vector storage (functional but not optimized)
   - Migrate to Vertex AI Vector Search for production-scale performance
   - Estimated: 768-dimension index, ~10K vectors initial capacity

2. **Reddit Direct API (PRAW)**
   - Setup instructions added to TASKS.md
   - Enables subreddit-specific queries, comment threads, real-time monitoring
   - Currently covered by Exa.ai semantic search - direct API is lower priority

3. **Additional Cultural Sources**
   - Perplexity integration for search + summarization
   - TikTok trend data via Exa/Tavily
   - Google Trends API for macro trend context

4. **Feedback Loop System**
   - Track which outputs Leo edits/approves
   - Feed corrections back into prompts
   - Evolution system already architected for this

### Business Considerations

1. **Rollout Strategy**
   - Phase 1: Leo as sole user (current)
   - Phase 2: Strategy team access
   - Phase 3: Account teams with appropriate guardrails

2. **Output Tiers**
   - Tier 1: Quick strategic bullets (seconds)
   - Tier 2: Full strategic write-up (minutes)
   - Tier 3: Complete presentation deck (minutes)

3. **Training Data Expansion**
   - Additional JL case studies and presentations
   - Historical pitch decks
   - Award submissions and results

4. **Cost Monitoring**
   - Claude API usage tracking
   - Embedding costs per document
   - Storage costs for vectors and exports

### Security & Compliance

1. **Data Classification**
   - Ensure client-sensitive data is properly handled
   - Consider per-client data isolation if needed

2. **Access Control**
   - Implement user authentication before broader rollout
   - Role-based permissions for different output types

3. **Audit Trail**
   - Log all generations for review
   - Track data lineage from source to output

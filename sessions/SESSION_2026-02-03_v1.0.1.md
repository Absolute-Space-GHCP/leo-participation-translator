## Key Design Decisions

### 1. Multi-Agent Architecture

**Decision:** Use specialized agents instead of monolithic app
**Rationale:**

- Each task requires different expertise
- Can test/refine independently
- Leo can guide strategic agent directly
- More maintainable

### 2. Knowledge Graph

**Decision:** Add knowledge graph alongside vector search
**Rationale:**

- Vector search finds similar content
- Knowledge graph finds **relationships**
- Enables queries like "What patterns counter negative perception?"

### 3. Task-Based Model Routing

**Decision:** Route tasks to optimal model by complexity
**Rationale:**

- Simple tasks (parsing, embedding) → cheap models
- Complex tasks (framework application) → Claude Opus 4.5
- Cost optimization without sacrificing quality

### 4. Google Slides Output

**Decision:** Google Slides instead of PPTX
**Rationale:**

- Native to JL's Google Workspace
- Real-time collaboration
- Leo can edit directly

### 5. Sanitized Adaptation

**Decision:** All code adapted from personal build has zero external references
**Rationale:**

- User plans to open-source personal projects
- Participation Translator code must be independent
